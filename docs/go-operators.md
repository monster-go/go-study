# Go 运算符：算术、比较、逻辑与位运算

> 知识点总结：掌握 Go 中算术、关系、逻辑、位运算与赋值运算符的语义；理解**整数除法**、**无隐式类型转换**、**短路求值**等语言特性，能写出正确的数值与布尔表达式。

---

## 1. 为什么需要了解这个

运算符是表达式的基础。新人写 Go 时经常在运算符上踩坑：

- 用 `7 / 3` 期望得到 `2.33`，实际得到 `2`（整数除法截断）
- 把 `int` 和 `float64` 直接相加，编译报错（Go 无隐式转换）
- 混淆 `&`（按位与）与 `&&`（逻辑与）
- 以为 `++` 是表达式，写成 `n = i++`（Go 中 `++`/`--` 只能是独立语句）
- 不理解 `&&`、`||` 的**短路（short-circuit）**行为，导致副作用重复执行

本篇建立在 [变量与常量](go-variables-types-and-constants.md) 之后——你已会基本类型与显式转换，现在学习如何用运算符组合它们。后续控制流、切片索引都会用到比较与算术运算。

## 2. 核心概念

### 2.1 运算符分类总览

| 类别 | 运算符 | 说明 |
|------|--------|------|
| 算术（arithmetic） | `+` `-` `*` `/` `%` | `+` 也可用于字符串拼接 |
| 关系（relational） | `==` `!=` `<` `>` `<=` `>=` | 结果为 `bool` |
| 逻辑（logical） | `&&` `\|\|` `!` | 仅用于布尔值；有短路 |
| 位（bitwise） | `&` `\|` `^` `&^` `<<` `>>` | 作用于整数类型 |
| 赋值（assignment） | `=` `+=` `-=` … | 复合赋值是简写 |
| 自增自减 | `++` `--` | **只能是独立语句**，不是表达式 |
| 取地址 / 解引用 | `&` `*` | 与指针配合，见 2.7 |

### 2.2 算术运算符

```go
a, b := 7, 3
a + b   // 10
a - b   // 4
a * b   // 21
a / b   // 2  ← 两个 int 相除，结果为 int，小数部分丢弃
a % b   // 1  ← 取余，仅整数类型
```

| 场景 | 行为 |
|------|------|
| 两个 `int` 相除 | 结果为 `int`，向零截断 |
| 需要小数 | 先显式转换：`float64(a) / float64(b)` |
| 字符串 `+` | 拼接，如 `"hello" + " " + "go"` |
| 不同类型数值运算 | **编译错误**，须先转换到同一类型 |

**整数除法如何得到结果（可复算）：** `7 / 3` 的数学商约为 2.33…，Go 规定整数相除**向零截断**（丢弃小数部分），因此结果是 `2`，不是四舍五入：

```
7 ÷ 3 = 2 余 1   →   7 / 3 = 2，7 % 3 = 1
```

**无隐式转换示例：**

```go
var i int = 1
var f float64 = 1.5
// i + f  // 编译错误：invalid operation
float64(i) + f // OK
```

### 2.3 关系运算符

比较两个**同类型可比较**的值，返回 `bool`：

```go
x == y   // 相等
x != y   // 不等
x < y    x > y    x <= y    x >= y
```

| 可比较 | 不可直接 `==` 比较 |
|--------|-------------------|
| 基本类型、指针、数组、结构体（字段可比较时） | slice、map、function（只能用 `== nil`） |

**slice / map 为何不能 `==`？**

slice、map、function 是**引用类型**，`==` 比较的是**是否指向同一块底层数据**，而不是内容是否相同。因此 `s1 == s2` 对 slice **编译不通过**；map 同理。它们**只能与 `nil` 比较**：

```go
var s []int
s == nil   // OK：判断 slice 是否为 nil

m := map[string]int{}
m == nil   // OK：空 map 字面量 `{}` 不是 nil，见下方说明
```

**比较内容的三种方式：**

| 方式 | 适用 | 说明 |
|------|------|------|
| `slices.Equal` / `maps.Equal` | Go 1.21+ | 标准库，类型安全，推荐 |
| `reflect.DeepEqual` | 任意可比较结构 | 通用但较慢，适合嵌套结构 |
| 手写循环 | 自定义规则 | 性能敏感或需特殊语义时用 |

**1. 标准库 `slices.Equal` / `maps.Equal`（推荐）**

```go
import (
    "maps"
    "slices"
)

s1 := []int{1, 2, 3}
s2 := []int{1, 2, 3}
slices.Equal(s1, s2) // true

m1 := map[string]int{"a": 1, "b": 2}
m2 := map[string]int{"b": 2, "a": 1} // map 无序，内容相同即可
maps.Equal(m1, m2)                   // true
```

**2. `reflect.DeepEqual`（通用）**

```go
import "reflect"

reflect.DeepEqual(s1, s2) // true
reflect.DeepEqual(m1, m2) // true

// 也适用于嵌套 slice、含 slice 字段的结构体等
type Config struct { Tags []string }
reflect.DeepEqual(Config{[]string{"go"}}, Config{[]string{"go"}}) // true
```

**3. 逐元素比较（手写）**

元素类型可比较时，可自己循环；map 需遍历 key 再比 value：

```go
// slice：长度 + 逐元素
func intSliceEqual(a, b []int) bool {
    if len(a) != len(b) {
        return false
    }
    for i := range a {
        if a[i] != b[i] {
            return false
        }
    }
    return true
}

// map：长度 + 逐 key 查 value
func stringIntMapEqual(a, b map[string]int) bool {
    if len(a) != len(b) {
        return false
    }
    for k, v := range a {
        if b[k] != v {
            return false
        }
    }
    return true
}
```

**注意：** `var s []int` 与 `s := []int{}` 都「长度为 0」，但前者是 `nil`、后者不是；`reflect.DeepEqual(nil, []int{})` 为 `false`。比较前想清楚要不要区分 nil 与空 slice。

### 2.4 逻辑运算符与短路求值

```go
ok && ready   // 两者都为 true 才 true
ok || backup  // 其一为 true 即 true
!ok           // 取反
```

**短路（short-circuit）：**

| 表达式 | 若左侧为 | 右侧 |
|--------|----------|------|
| `A && B` | `false` | **不执行** B |
| `A \|\| B` | `true` | **不执行** B |

```go
if user != nil && user.IsAdmin() { ... }
// user 为 nil 时不会调用 IsAdmin()，避免空指针 panic
```

### 2.5 位运算符

作用于**整数类型**的底层二进制位。Go 在内存里用补码二进制表示整数；位运算就是**逐位**对两个数的每一位做规则运算。

#### 十进制 ↔ 二进制（以 `12` 和 `10` 为例）

先把十进制写成二进制（从右往左，每位是 2 的幂）：

| 十进制 | 二进制 | 怎么算 |
|--------|--------|--------|
| 12 | `1100` | 8 + 4 = 12（第 3、2 位为 1） |
| 10 | `1010` | 8 + 2 = 10（第 3、1 位为 1） |

位权对照（4 位示例）：

```
位权：  8   4   2   1
12：    1   1   0   0
10：    1   0   1   0
```

#### 按位与 `&`、按位或 `|`、按位异或 `^`

三个运算符都是**对齐同一位**再运算：

| 运算符 | 规则（同一位上） |
|--------|------------------|
| `&` 与 | 两个都是 1 → 1，否则 → 0 |
| `\|` 或 | 有一个是 1 → 1，都是 0 → 0 |
| `^` 异或 | 相同 → 0，不同 → 1 |

以 `12 & 10` 为例，逐位列对齐：

```
  12 = 1100
& 10 = 1010
  ─────────
   8 = 1000   ← 只有第 3 位两边都是 1
```

| 位（从右数） | 12 | 10 | `&` | `\|` | `^` |
|-------------|----|----|-----|------|-----|
| 第 0 位（1） | 0 | 0 | 0 | 0 | 0 |
| 第 1 位（2） | 0 | 1 | 0 | 1 | 1 |
| 第 2 位（4） | 1 | 0 | 0 | 1 | 1 |
| 第 3 位（8） | 1 | 1 | 1 | 1 | 0 |
| **结果二进制** | | | `1000` | `1110` | `0110` |
| **换回十进制** | | | **8** | **14** | **6** |

因此：`12 & 10 = 8`，`12 | 10 = 14`，`12 ^ 10 = 6`。

#### 按位清除 `&^`（Go 特有，AND NOT）

`a &^ b`：看 **b** 的每一位——若 b 该位为 1，结果该位强制为 0；若 b 该位为 0，结果该位保留 **a** 的原值。

```
  12 = 1100
&^10 = 1010   （b 为 1 的位「清掉」a 对应位）
  ─────────
   4 = 0100
```

第 1 位 b=1 → 清 0；第 3 位 b=1 → 清 0；其余位保留 a → `0100` = 4。

#### 移位 `<<` 与 `>>`

| 运算符 | 含义 | 示例推导 |
|--------|------|----------|
| `<<` 左移 | 所有位向左移 n 位，右侧补 0，相当于 × 2ⁿ | `1 << 3`：`0001` → `1000` = 8 |
| `>>` 右移 | 所有位向右移 n 位，相当于 ÷ 2ⁿ（整数向零截断） | `8 >> 2`：`1000` → `0010` = 2 |

#### 运算符速查表

| 运算符 | 含义 | 示例 `12 op 10` | 结果 |
|--------|------|-----------------|------|
| `&` | 按位与 | 见上表 | 8 |
| `\|` | 按位或 | 见上表 | 14 |
| `^` | 按位异或 | 见上表 | 6 |
| `&^` | 按位清除 | 见上 | 4 |
| `<<` | 左移 | `1 << 3` | 8 |
| `>>` | 右移 | `8 >> 2` | 2 |

日常业务代码较少手写位运算；读权限常量（`iota` 左移，见 [变量与常量](go-variables-types-and-constants.md)）、协议解析时会遇到。

### 2.6 赋值与复合赋值

```go
n = 10
n += 3   // n = n + 3
n -= 1
n *= 2
n /= 2
n %= 3
```

复合赋值要求左右类型兼容；`n` 必须先声明。

### 2.7 自增、自减与指针运算符

**`++` 和 `--` 只能是独立语句：**

```go
i++      // OK
++i      // 语法错误：Go 没有前缀 ++
j = i++  // 语法错误：++ 不是表达式
```

**取地址 `&` 与解引用 `*`：**

```go
n := 42
p := &n   // p 是指向 int 的指针
*p = 100  // 通过指针修改 n
```

指针运算细节在后续专题展开；此处知道 `&` 取地址、`*` 解引用即可。

### 2.8 运算符优先级（简记）

从高到低（同一行优先级相近）：

1. `*` `/` `%` `<<` `>>` `&` `&^`
2. `+` `-` `|` `^`
3. `==` `!=` `<` `<=` `>` `>=`
4. `&&`
5. `||`

拿不准时用**括号**明确意图，可读性优先于省字符。

## 4. 常见坑与排查

### 4.1 整数除法丢失小数

```go
avg := sum / count // 两个 int → 截断
```

**修复：** `float64(sum) / float64(count)`，或全程用浮点类型。

### 4.2 混淆按位与与逻辑与

```go
if a & b > 0 { }   // 先算 a&b，再比较
if a && b { }      // 逻辑与，要求 a、b 为 bool
```

**修复：** 布尔条件用 `&&`；位掩码用 `&`。

### 4.3 对 slice 使用 `==`

```go
s1 := []int{1, 2}
s2 := []int{1, 2}
// s1 == s2  // 编译错误
```

**修复：** 用 `slices.Equal`（Go 1.21+）或 `reflect.DeepEqual`，或自己循环比较。

### 4.4 副作用在短路表达式中「有时不执行」

```go
if ok || doSomething() { } // ok 为 true 时 doSomething 不会被调用
```

**修复：** 若必须执行副作用，拆成独立语句，不要依赖 `||` 右侧。

## 5. 小结与延伸阅读

**要点回顾：**

1. Go **无隐式数值转换**，混合类型运算须显式 `float64(i)` 等
2. 两个整数 `/` 结果为整数，小数被截断
3. `&&`、`||` 会短路，常用于 `nil` 检查后再访问字段
4. `++`/`--` 只能是独立语句，不是表达式
5. slice、map 不能 `==` 比较内容（仅可与 `nil` 比）
6. 位运算 `&` `|` `^` `<<` `>>` 用于整数；与逻辑运算符不同
